# Release 0.7.0 - Constellation Graph

## Why This Release Exists

The graph should help users understand relationship density and direction without becoming a decorative star field.

## User Problem

Vault graphs can become visually noisy. Users need clearer state, contrast, and guidance for interpreting relationships.

## Brand Alignment

Constellation language fits the theme when it clarifies orbits, clusters, and relationships. It must remain practical and readable.

## Public Hook

Make vault relationships feel more readable, intentional, and alive.

## Scope

- Improve graph styling where it supports readability.
- Document tag or role conventions for graph use.
- Add graph visual QA notes or screenshots.
- Keep graph changes compatible with dark and light modes.

## Non-Goals

- Do not implement graph analytics.
- Do not claim automated relationship classification.
- Do not add remote assets or heavy visual effects.

## Files Expected To Change

- `styles/graph.css`
- `docs/theme-guidelines/`
- `README.md`
- `CHANGELOG.md`
- `manifest.json`
- `package.json`
- `versions.json`
- `theme.css`

## QA Plan

- Run `npm run build`.
- Run `npm run check`.
- Run `git diff --check`.
- Verify local and global graph readability in dark and light modes.
- Verify hover, focus, and current-node states.

## Release Notes Draft

Improve graph readability and document constellation-style conventions for interpreting relationships across a vault.

## Risks

- Graph CSS can be brittle across Obsidian versions. Prefer Obsidian variables and conservative selectors.
