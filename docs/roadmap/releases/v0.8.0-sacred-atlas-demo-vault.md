# Release 0.8.0 - Sacred Atlas Demo Vault

## Why This Release Exists

Users should be able to inspect a working example instead of assembling the full system from isolated docs.

## User Problem

The theme is easiest to understand when notes, Canvas maps, role cues, and graph conventions are visible together.

## Brand Alignment

The demo vault should show the theme as an operating map for projects and knowledge evolution. It must be useful, sanitized, and light enough to ship with the repo.

## Public Hook

Explore a working example vault built around systems mapping, project clarity, and knowledge evolution.

## Scope

- Add a small demo vault or example pack.
- Include sample notes, Canvas maps, callouts, and graph/tag conventions.
- Add a README walkthrough.
- Keep assets small and local.

## Non-Goals

- Do not include private or customer data.
- Do not ship a large vault.
- Do not imply that SacraMap exists in this theme repo.

## Files Expected To Change

- `examples/` or `demo-vault/`
- `README.md`
- `CHANGELOG.md`
- `manifest.json`
- `package.json`
- `versions.json`
- `theme.css` if CSS source changes

## QA Plan

- Run `npm run build`.
- Run `npm run check`.
- Run `git diff --check`.
- Open the demo pack in a clean vault.
- Verify no private data, heavy assets, or remote references are included.

- Example content can become stale. Keep it small and tied to the documented role language.
