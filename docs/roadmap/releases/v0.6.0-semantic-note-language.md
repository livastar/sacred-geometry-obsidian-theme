# Release 0.6.0 - Semantic Note Language

## Why This Release Exists

The theme's role language should help normal notes, not only Canvas maps.

## User Problem

Users move between notes, Canvas, and backlinks. If semantic cues only exist in Canvas, the system language breaks during everyday writing.

## Brand Alignment

Mission, decision, knowledge, outcome, and flow cues should become a readable note grammar that supports thinking without turning notes into decoration.

## Public Hook

Bring mission, decision, knowledge, and outcome cues into everyday notes.

## Scope

- Add note templates or examples for semantic note patterns.
- Document frontmatter and callout conventions.
- Improve note-facing callout examples if needed.
- Preserve current Canvas role names.

## Non-Goals

- Do not create a plugin or parser.
- Do not require users to adopt a strict vault taxonomy.
- Do not claim automated system understanding.

## Files Expected To Change

- `templates/`
- `docs/theme-guidelines/`
- `README.md`
- `CHANGELOG.md`
- `manifest.json`
- `package.json`
- `versions.json`
- `theme.css` if CSS source changes

## QA Plan

- Run `npm run build`.
- Run `npm run check`.
- Run `git diff --check`.
- Verify reading view, Live Preview, and source mode remain readable in dark and light modes.
- Verify semantic callouts and templates contain no private data.

## Release Notes Draft

Add a semantic note language that brings the theme's mission, decision, knowledge, and outcome cues into everyday writing workflows.

## Risks

- Over-prescriptive note conventions can reduce usefulness. Keep patterns optional and adaptable.
