# Release 0.5.0 - Living Map Templates

## Why This Release Exists

Users should not start from a blank Canvas when they want to map a project, agent ecosystem, decision path, knowledge atlas, flow network, or resource map.

## User Problem

Canvas is powerful but open-ended. People need opinionated starting patterns that show how Sacred Geometry Systems uses structure, roles, and flow.

## Brand Alignment

This release turns geometry into a practical map grammar. The templates should reveal origins, boundaries, relationships, and movement without adding unsupported plugin behavior.

## Public Hook

Start from guided Canvas patterns for projects, agents, decisions, knowledge, and feedback loops.

## Scope

- Add separate guided Canvas templates.
- Document when to use each template.
- Keep existing semantic Canvas role names stable.
- Add visual QA evidence for the new templates.

## Non-Goals

- Do not implement SacraMap.
- Do not add automated layout, infinite zoom, or live playback claims.
- Do not change the public role vocabulary unless a later major release accepts that break.

## Files Expected To Change

- `canvas/` or `canvas/templates/`
- `README.md`
- `CHANGELOG.md`
- `docs/theme-guidelines/canvas-project-ecosystem.md`
- `manifest.json`
- `package.json`
- `versions.json`
- `theme.css` if CSS source changes
