# Living Map Templates

These optional Canvas templates provide focused starting points for common Obsidian systems-mapping workflows. Copy the `.canvas` files into a vault and adapt the node text to your project.

## Templates

| Template | Use When | Core Pattern |
| --- | --- | --- |
| `Agent Ecosystem.canvas` | You need to map people, automations, tools, and handoffs. | Mission -> core system -> agents -> interaction -> outcome. |
| `Project Constellation.canvas` | You need a project overview that keeps workstreams, decisions, and delivery connected. | Mission -> project spine -> workstreams -> release gate -> outcome. |
| `Decision Tree.canvas` | You need to compare options and leave a clear decision record. | Purpose -> choice point -> options -> review -> outcome. |
| `Knowledge Atlas.canvas` | You need to organize sources, concepts, evidence, gaps, and synthesis. | Inquiry -> topic center -> evidence and concepts -> reusable knowledge. |
| `Flow Network.canvas` | You need to trace movement of work, data, requests, or feedback. | Trigger -> process -> agent -> gate -> delivery loop. |
| `Resource Map.canvas` | You need to organize tools, references, assets, owners, and readiness. | Purpose -> resource hub -> resource clusters -> readiness gate -> reusable kit. |

## Usage Notes

- Keep node summaries short enough to read at overview scale.
- Use the same semantic roles documented in `docs/theme-guidelines/canvas-project-ecosystem.md`.
- Enable `snippets/SG Canvas Cards.css` when you want role-based Canvas card styling in addition to the base theme.
- Do not add private content before sharing screenshots or templates publicly.
