# Sacred Geometry Systems Brand Book

Sacred Geometry Systems is a dark Obsidian theme language for turning complex work into living maps. The system uses sacred-geometry forms as visual grammar: not as mysticism, but as a precise way to show origin, intersection, progression, feedback, and intelligence.

Tagline: Geometry is frozen music. Systems are living symphonies.

## Core Intention

- Reveal hidden structure.
- Create human zoom-out / zoom-in clarity.
- Turn complexity into living maps.
- Design systems with meaning, rhythm, and coherence.

The tone is precise, contemplative, and alive. Interfaces should feel like an instrument panel for complex thought: calm enough to read, luminous enough to signal movement, and structured enough to make relationships obvious.

## Sacred Geometry Language

| Form | Meaning | Obsidian use |
| --- | --- | --- |
| Vesica Piscis | Intersection, co-creation | Consent gates, partnerships, shared ownership |
| Seed of Life | Origin, ideation | New projects, first principles, starting questions |
| Flower of Life | Ecosystem, interconnection | Agent ecosystems, knowledge clusters, dependency maps |
| Metatron's Cube | Architecture, intelligence | Core system logic, routing, orchestration |
| Fibonacci Spiral | Natural growth, progression | Roadmaps, learning loops, rollout paths |
| Torus | Feedback loop, living flow | Health checks, retrospectives, signal circulation |

Geometry should clarify structure before it decorates. Use lines, rings, grids, and glows to reveal relationships, not to add visual noise.

## Visual System

| Token | Hex | Role |
| --- | --- | --- |
| Obsidian | `#03080d` | Primary background |
| Ivory | `#f5ead6` | Primary text and highlight contrast |
| Gold | `#d8a84f` | Sacred structure, focus, priority |
| Copper | `#b87345` | Boundaries, warmth, paid product cues |
| Deep Indigo | `#12122d` | Nested panels and quiet depth |
| Signal Cyan | `#16c7e8` | Data flow, activity, live state |

Typography pairs geometric-sans headings with clean humanist body text. In CSS, that means system-first sans stacks that stay fast and reliable inside Obsidian:

- Headlines: geometric sans, uppercase-friendly, measured and calm.
- Body: clean humanist sans, readable at Canvas and note scale.
- Code: compact monospace with dark-panel contrast.

Cards use tight radii, thin luminous borders, and dark panel surfaces. Glow is reserved for state: active flow, focused cards, live activity, or priority.

## Obsidian Canvas Use

Canvas maps should expose nested structure at a glance:

- Nested cards show system layers without hiding the parent context.
- Templates give repeatable starting points for common map shapes.
- Agent roles sit on the outer operating ring.
- Flow tracing follows cyan data/task movement through the system.
- Knowledge clusters use gold or indigo groupings for memory, insights, and sources.
- Decision paths use copper or gold links to mark choice points and outcomes.

The default starter map is organized around a mission node, a core system, agent roles, resources, interactions, decisions, outcomes, and knowledge clusters.

## Live Maps & Animation Principles

- Pulse means activity.
- Glow means priority.
- Orbit means relationship.
- Flow means data or task movement.
- Zoom means context shift.
- Playback means evolution over time.

Motion must remain optional and useful. When `prefers-reduced-motion` is enabled, the theme should preserve state through color, border, and contrast instead of animation.

## Template Library

The template vocabulary supports these repeatable map types:

- Agent Ecosystem
- Project Constellation
- Decision Tree
- Knowledge Atlas
- Flow Network
- Resource Map

Each template should help a user move between overview and detail without losing the map's center of gravity.

## SacraMap Plugin Concept

SacraMap is a future commercial plugin concept, not an implemented plugin in this repository.

Concept features:

- Infinite zoom canvas.
- Sacred-geometry layout engine.
- Animated live maps.
- Nested cards and clusters.
- Overview-to-detail navigation.
- System health and flow states.
- Presentation mode.
- Commercial plugin for complex thinking.

The theme should make SacraMap feel plausible by providing the visual language now, while keeping plugin behavior out of scope until a separate implementation exists.
